Portfolio Project Overview

rag-eval

An offline-first evaluation harness for RAG question-answering systems. It pools SQuAD v2 paragraphs into a tiny corpus, retrieves top-k docs, generates only from retrieved context, and flags retrieval or hallucination regressions.

Python 3.11+ uv install workflow Offline mock RAG Regression comparison Portfolio-ready docs

Quick Start With uv

Install dependencies, run the k=5 vs k=1 regression demo, and execute tests:

uv sync --dev
uv run python -m example_rag demo
uv run pytest -q

RAG Evaluation Loop

The harness treats the RAG application as a pluggable system under test: retrieve docs, generate an answer, score the output, save the run, and compare it against a baseline.

Module Map

The harness (rag_eval) and the swappable system under test (example_rag: retrievers, generators, and the CLI) are separate packages. The harness depends only on the Retriever/Generator protocols and never imports the concrete system, so each module has a narrow job and a new system plugs in without touching the evaluator.

Metric Snapshot

The demo runs the same eval at k=5 and k=1 (only k changes). Real SQuAD v2 numbers; retrieval hit-rate moves the most.

Regression Logic

`compare_runs` loads two JSON artifacts, computes candidate minus baseline for each aggregate metric, and flags any score that drops beyond the threshold.

Data

  • data/squad_sample.jsonl: 50 committed real SQuAD v2 answerable questions.
  • data/squad_corpus.jsonl: 235 pooled paragraphs (mostly distractors).
  • data/squad_subset.jsonl: local HuggingFace cache after online download.

Extension Points

  • EmbeddingRetriever: connect real embeddings or a vector DB.
  • LLMGenerator: connect a provider-backed generation call.
  • LLMJudgeScorer: add semantic judgment later.

Portfolio Pitch

This project is quality infrastructure for RAG apps. It proves you understand not only retrieval and generation, but also how to measure regressions before users see hallucinated or unsupported answers.
